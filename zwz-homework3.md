### 5.4 黑盒测试方法说明

本次测试主要采用黑盒测试方法，不关注系统内部代码实现，只根据输入条件和页面反馈判断功能是否满足需求。

| 方法         | 应用说明                                                                                                 |
| ------------ | -------------------------------------------------------------------------------------------------------- |
| 等价类划分法 | 将用户状态划分为已登录、未登录；将库存状态划分为有库存、无库存；将活动状态划分为未开始、进行中、已结束。 |
| 边界值分析法 | 针对秒杀开始时间、结束时间、库存数量为0和库存数量为1等边界情况进行测试。                                 |
| 场景法       | 模拟用户从进入商品详情页、点击秒杀按钮、提交订单、查看结果的完整业务流程。                               |

### 5.9 测试结果与建议

本次测试共设计并执行12条测试用例，其中10条通过，2条不通过，用例成功率为83.33%。测试结果表明，《高并发电商秒杀平台》的秒杀下单模块能够完成登录校验、秒杀时间校验、库存校验、正常下单和重复下单限制等核心功能。

建议后续对系统进行以下优化：

1. 库存不足、活动结束、重复下单等失败场景应给出更明确的提示；
2. 秒杀按钮在用户点击后应短暂禁用，避免用户重复点击造成重复请求；
3. 对高并发场景下的库存一致性和接口稳定性进行进一步压力测试。

## 六、实验小结

通过本次系统功能测试实验，我进一步理解了功能测试的基本流程。系统功能测试不仅要验证正常流程是否可用，还要关注异常输入、边界条件和用户操作反馈。

本次实验中，我选择《高并发电商秒杀平台》的秒杀下单模块作为测试对象，先进行测试需求分析和测试点提取，再使用等价类划分法、边界值分析法和场景法设计测试用例，最后执行测试并记录缺陷。通过测试发现，该模块核心功能基本满足要求，但在失败提示和按钮防重复点击方面仍存在改进空间。

本次实验让我认识到，测试工作不能只关注“功能能否跑通”，还要关注用户在异常情况下能否获得清晰反馈。后续开发系统时，应在功能设计阶段同步考虑测试点和异常处理，从而提升系统质量和用户体验。
